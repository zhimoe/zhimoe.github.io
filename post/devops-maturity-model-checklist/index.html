DevOps 能力成熟度模型

2019-07-24 编程 #devops

之前听说过 AWS 的软件工程师是需要自己写需求说明书，前后端代码，测试和上线。还有 instagram 的工程师可以做到 python 的代码提交如果合并到主分支后可以在一个小时内自动部署到生产被用户使用到，感觉这个非常神奇。如果需要做到这个，对组织级与个人都有极高的 devops 能力成熟度要求。
上周代表 CRM 项目通过了信通院的 DevOps 三级认证。感觉提升的空间很大。专门看了一下信通院发布的成熟度模型标准。

核心要点是要有统一的管理系统，系统之间需要联动，
例如记录故事的系统，如何和你提交记录关联？
测试的缺陷问题如何和你的故事关联？
生产正在运行的代码，如何和代码库的某个基线对应上？
测试报告/需求说明书是否统一管理并和你的迭代有关联？
是否可以做到事故之后快速实现回滚部署？
